# Extras – Additional Lab Implementations

---

## 🎯 Objective

Extend the core lab environment with additional technologies and configurations that fall outside the original project scope.

This section documents standalone implementations that emerged from curiosity, real-world relevance, or challenges encountered during the lab. Each extra is self-contained and can be implemented independently at any point during the lab.

Unlike the structured phase-based design of the core project, the extras section has no defined order or dependency chain. It is an open-ended collection of additional scenarios that complement the existing infrastructure and expand the technology coverage of the lab.

---

## 📂 Extras

- `E01-vpn-wireguard.md` – WireGuard VPN server deployment and remote access configuration
- `E02-action1-patch-management.md` – Cloud-based patch management with Action1

---

## 🔐 Design Principles

Regardless of their standalone nature, all extras follow the same design philosophy as the core lab:

- Implementations reflect real-world enterprise practices where possible.
- Role separation and security considerations are maintained.
- Decisions and trade-offs are documented and justified.
